位元系列各時代特性

16位元圖形化使用者介面

  • 具有了許多典型的作業系統的功能，包括擁有自己的執行檔格式以及為應用程式提供自己的裝置驅動程式（計時器、圖形、印表機、滑鼠、鍵盤以及音效卡）
  • 透過協作式多工允許使用者在同一時刻執行多個圖形應用程式
  • 實作了一個設計精良的、基於段的軟體虛擬記憶體方案，使其能夠執行大於物理記憶體的應用程式。程式碼段和資源在記憶體不足的時候進行交換，並且當一個應用程式釋放處理器控制時，特別是等待使用者輸入的時候，資料段會被移入記憶體

16位元和32位元混合作業系統

  • 仍然需要依賴16位元的DOS基層程式才能執行，不算是真正意義上的32位元作業系統
  • 使用DOS程式碼，架構也與16位元DOS一樣，核心屬於單核心，但也引入了部份32位元作業系統的特性
  • 包括Windows 95（第一版發行於1995年，後來的改進版本發行於1996年和1997年），以及三個版本的Windows 98（1998年、1999年和2000年）

32位元作業系統

  • 真正的純32位元作業系統。
  • 核心採用混合式核心即改良式微核心。
  • 它們包括 Windows NT 3.1（1992年）、 NT 3.5、 NT 3.51、 NT 4.0、 Windows 2000、 Windows XP、32位元 Windows Vista和32位元 Windows 7、 Windows 8、 Windows 8.1（Windows XP、Windows Vista、Windows 7、Windows 8、Windows 8.1同時有32位元的x86版本與64位元的x64版本

64位元作業系統

  • 微軟曾對兩種不同的64位元架構提供支援，其一是Intel公司和HP聯合開發具有革新化的Itanium家族架構，或稱之為IA-64；和AMD公司開發的演進化的x86-64架構
  • 兩種架構的核心設計思想不同，因此兩種架構的操作系統和應用軟體不具有互通性，但都對傳統的IA-32架構的軟體一定程度上提供支援，微軟在發布Windows Server 2012 R2前放棄了對Itanium架構的支援。現在微軟的64位元產品指的單單是x86-64架構
  • 支援x64架構的Windows產品有：
    • Windows XP Professional x64 Edition
    • Windows Server 2003/2003R2全線產品（Web版除外）
    • Windows Vista/7/8/8.1
    • Windows Serverh3 2008/2008R2/2012/2012R2全線產品
    • Windows 10

中文支援

早期IBM PC並不支援中文處理，台灣開為使用DOS個人電腦，編寫能處理中文系統，著名建基於DOS的中文系統有國喬 、倚天、震漢等等。由於是用擷取繪寫螢幕指令的方式，截取畫面上的文字，將兩個ASCII編碼改為以圖形顯示的大五碼中文字，導致中文文書處理相當複雜（例如換行、左右對齊）。承繼自DOS的固定行列顯示、早期只能顯示16x15圖素的點陣字體，早期的中文系統只能算是勉強能於個人電腦上處理中文訊息。隨著繁中和簡中版的Windows 95推出，雖然倚天中文系統已支援SVGA模式以24x24圖素點陣字型顯示、使用平滑字型、震撼中文系統已支援使用TrueType字列印等，但各種基於DOS的中文系統，於Windows 95的普及下，很快便消失。

近代產品比較

Windows XP：

  • 更快的啟動與休眠過程
  • 提供驅動程式回復功能以應對由於更新或升級裝置驅動程式可能造成的問題
  • 提供更加友好的用戶介面，以及為桌面環境開發主題的架構
  • 快速切換使用者，允許一個使用者儲存當前狀態及已開啟的程式，同時允許另一使用者在不影響該等資訊的情況下登入
  • ClearType字體渲染機制，用以提高液晶顯示器上的文字可讀性
  • 遠端桌面功能允許使用者通過網路遠程連接一台運行Windows XP的機器操作應用程式、文件、印表機和裝置
  • 支援多數DSL數據機以及無線網路連接，以及通過火線和藍芽的網路連接

Windows 7：

  • 是Windows Vista的「改良版」
  • 提高了螢幕觸控支援和手寫辨識，支援虛擬硬碟，改善多核心處理器的運作效率，開機速度和核心改進。
  • 支援多個顯示卡、新版本的Windows Media Center: 一個供Windows Media Center使用的桌面小工具、增強的音訊功能、內建的XPS和Windows PowerShell
  • 原有的「快速啟動」功能已被「釘」在工具列上的縮小程式圖示取代。這個功能不僅具有快速啟動原有的功能，也同時具有顯示目前正在運行的程式圖示的功能
  • 刪除了Windows Vista中的許多功能和一些程式，如傳統開始功能表樣式、Windows Ultimate Extras、筆跡球和Windows行事曆。Windows 7去除了Windows Vista中捆綁的三個軟體—Windows Photo Gallery、Windows Movie Maker和Windows Mail，以名為Windows Live Essentials的可選軟體包方式單獨提供
  • Windows 7涵蓋32位元和64位元二個版本，顧及從32位元系統過渡到64位元系統的趨勢

Windows 8：

  • 採用與Windows Phone 8相同的NT核心，被認為是微軟反擊主導平板電腦及智慧型手機作業系統市場的蘋果iOS和Google Android的作業系統
  • 備微軟適用於筆記型電腦和桌上型電腦平台的傳統視窗系統顯示方式外，還特別強化適用於觸控螢幕的平板電腦設計
  • Windows 8共支援109種語言，相比Windows 7增加14種，可以為全球超過45億的使用者提供母語版本
  • 採用由安德斯·海爾斯伯格領導，基於Apache 2.0開源授權協議，開源於CodePlex的編寫語言TypeScript，TypeScript是JavaScript的超集，是套用級JavaScript開發語言，可編譯成基於標準的JavaScript，為大團隊和大規模專案提供更好的工具來開發客戶端、伺服器端/雲端等套用，提供類等特性的支援。
  • 因為為了行動方面而做出的改變過快，導致桌面使用者不習慣，加上雖然已有Metro介面存在，但是工作時卻仍然要經常切換到桌面，而使用者的困擾通常是沒有開始按鈕、沒有鬧鐘、搜尋功能不好用、Metro介面和桌面不搭、沒有Windows Aero毛玻璃特效
  • 新的資源管理器支援右單擊加載ISO、IMG 和VHD 檔案。而Windows 7 僅支援通過磁盤管理器或命令列介面加載VHD
  • Windows 8 中內置了安全防護軟體Windows Defender，內核和介面和Microsoft Security Essentials相同。旨在提高系統安全性，查殺惡意軟體與引導啟動的惡意軟體。8.Windows 7 軟體都可以直接在 Windows 8 上應用
  • 個人化設定可以在多台 Windows 8 裝置間同步
  • 放棄部分 Windows Aero 效果如 Aero Glass (毛玻璃效果) 等，整體感覺會失去立體感，但是其他如DWM和Peek 預覽仍然存在.；放棄開始功能表以及開始按鈕(開始按鈕已於Windows 8.1 重新出現)

Windows 8.1：

Windows 8.1（僅限於64位元版本）是微軟公司的第一款完整支援48位元虛擬記憶體訪存的作業系統，此前的x64位元版本僅僅達到對44位元的虛擬記憶體空間的支援。為了達到這個特性，依據微軟體作業系統的內部實現機制，CMPXCHG16b指令做為處理必先支援的前提。虛擬記憶體空間的擴展並不破壞原有x64位元程式的相容性。

開發中：Windows 10

  • 恢復了類似Windows 7的傳統開始按鈕，但是在細節上糅合了Windows 8和Windows 8.1的微軟「Modern瓷磚」設計
  • 將覆蓋所有尺寸和品類的Windows裝置，可無縫執行於微型電腦（類似英特爾伽利略）、手機（ARM晶片）、平板（ARM和x86晶片）、二合一裝置、桌面電腦及伺服器等幾乎所有硬體
